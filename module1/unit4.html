Programming Language Concepts

Q1: What is ReDOS and what part do ‘Evil Regex’ play?
A1: ReDOS is a regular expression denial of service attack which uses the fact that Regex may hang for a long amount of time when receiving an input string which takes a long time to iterate upon, leading to the program hanging for a long time. "Evil Regex" is a pattern which accepts strings of very long or indeterminate length, allowing for ReDOS attacks to be performed. Evil Regex can either be part of the program, or alternatively be injected by a malicious user (Weidman, n.d.).

Q2: What are the common problems associated with the use of regex? How can these be mitigated?
A2: Vulnerability to attacks such as ReDOS is one issue, which can be mitigated by avoiding the Evil Regex pattern (through experience and using various software), and preventing string injection via escaping characters.
Another issue is the inherent complexity of writing Regex strings, which can be mitigated using software such as ACRE which checks for errors (Larson, 2018).

Q3: How and why could regex be used as part of a security solution?
A3: Regex could be used for scanning large logs for specific information which pertains to security threats.

References:
Larson, E. (2018) Automatic Checking of Regular Expressions. 18th IEEE International Working Conference on Source Code Analysis and Manipulation (SCAM).
Weidman, A. (n.d.) Regular expression Denial of Service - ReDoS.

Recursion

This task involved creating a Python implementation of Towers of Hanoi along with an algorithm for it to be solved optimally. This can be found here.
Disks are represented by numbers, the smaller the number the smaller the disk. Users are not allowed to play, but can choose to either step manually through each turn in the solution, or allow the program to step automatically. The user can pick the amount of disks used in the simulation.

Q: What is the (theoretical) maximum number of disks that your program can move without generating an error? What limits the number of iterations? What is the implication for application and system security?
A: The maximum number of disks is 985 because Python will hit its default recursion limit of 1000. Python limits recursions in order to prevent overflow errors and attacks. In order to prevent the error however, I have set a maximum number of disks which can be inputted.

References:
Cormen, T. & Balkcom, D. (n.d.) Khan Academy: Towers of Hanoi.
